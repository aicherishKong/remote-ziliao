## 三、阶段一：基础入门 (3 小时)

### 3.1 概念建立 (30 分钟)

**核心定义**:
Agent Skills 是一个可被 Agent 在运行时发现、理解、调用和组合的能力模块。

**类比理解**:
- Agent 就像手机
- Skill 就像 App
- 生态就像应用商店

**为什么需要 Agent Skills?**
- 痛点: AI 不知道怎么做复杂任务
- 解决: 通过 Skill 提供结构化的能力模块
- 价值: 可发现、可执行、可复用

### 3.2 痛点理解 (30 分钟)

**真实场景**:
场景1: 补充单元测试
- 传统方式: 每次都要重新解释需求
- 使用 Skill: 一句话触发，自动执行完整流程

场景2: 代码审查
- 传统方式: AI 可能遗漏关键检查项
- 使用 Skill: 按照标准化流程执行

**对比方案**:
- Prompt: 一次性指令，难以复用
- Function Calling: 代码级调用，需要编程
- Agent Skills: 结构化能力，易于创建和复用

### 3.3 核心特性 (1 小时)

**特性1: 可发现**
- Agent 能知道这个 Skill 适合什么任务
- 通过 description 字段实现
- 示例: [展示 frontmatter]

**特性2: 可执行**
- Skill 内部有结构化流程
- 通过 Workflow 字段实现
- 示例: [展示 Workflow]

**特性3: 可复用**
- 同一份 Skill 可以跨场景使用
- 通过独立模块设计实现
- 示例: [展示复用场景]

### 3.4 术语体系 (30 分钟)

**核心术语表**:
1. **Skill**: 能力模块
2. **frontmatter**: 元数据配置
3. **body**: Skill 主体内容
4. **Workflow**: 执行流程
5. **Constraints**: 约束条件
6. **resources**: 资源目录
7. **Agent**: 智能体
8. **路由**: Skill 匹配机制
9. **注入**: Prompt 注入机制
10. **生态**: Skill 来源和分享

## 使用说明

这些示例展示了完整学习路径的标准结构和内容深度。在实际使用时：

1. **根据用户需求调整深度**: 快速入门 vs 系统掌握 vs 深度掌握
2. **根据技术特点调整结构**: 不同技术的三个视角侧重点不同
3. **根据应用场景调整案例**: 选择与用户场景相关的案例
4. **保持结构一致性**: 所有学习路径都应该遵循相同的章节结构
