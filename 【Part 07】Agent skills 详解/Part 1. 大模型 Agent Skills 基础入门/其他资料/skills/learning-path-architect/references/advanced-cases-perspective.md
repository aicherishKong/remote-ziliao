# Advanced Cases Perspective (进阶案例视角)

## 目标
通过真实案例理解技术的应用场景和最佳实践。

## 分析维度

### 1. 典型场景
- 这个技术最常用于哪 3-5 个场景？
- 每个场景的特点和适用条件
- 场景之间的差异和选择依据

### 2. 案例选择
- 每个场景选择一个代表性案例
- 案例应该具有实际应用价值
- 案例复杂度应该适中，既不过于简单也不过于复杂

### 3. 技术深化
- 每个案例涉及哪些进阶知识点？
- 这些知识点如何在案例中体现？
- 知识点之间的关联和依赖关系

### 4. 坑点预警
- 每个场景的常见错误和解决方案
- 为什么会出现这些错误？
- 如何避免和快速定位问题？

### 5. 性能优化
- 如何在实际应用中优化性能？
- 性能瓶颈通常在哪里？
- 优化的优先级和权衡

### 6. 工程实践
- 如何在生产环境中使用？
- 需要注意哪些工程化问题？
- 监控、日志、容错等实践要点

## 输出格式

```
进阶案例路径:
场景1: [场景名称]
  - 案例: [具体案例描述]
  - 涉及知识点: [知识点1, 知识点2]
  - 常见坑点: [坑点 + 解决方案]
  - 优化方向: [性能优化建议]

场景2: [场景名称]
  ...
```

## 设计原则

- **场景驱动**: 以实际应用场景为主线
- **问题导向**: 聚焦于解决实际问题
- **最佳实践**: 提供经过验证的解决方案
- **坑点预警**: 主动提示常见错误和解决方法
- **性能意识**: 关注性能优化和工程实践
