# Resource Recommendation (学习资源推荐)

## 目标
为每个学习阶段推荐高质量的学习资源。

## 执行步骤

### 1. 推荐官方文档
- 官方文档的必读部分
- 哪些章节是核心内容？
- 哪些章节可以跳过或选读？
- 官方文档的阅读顺序建议

**推荐格式**:
```
官方文档: [文档名称]
- 链接: [URL]
- 必读章节: [章节1, 章节2, ...]
- 选读章节: [章节3, 章节4, ...]
- 阅读顺序: [建议的阅读路径]
```

### 2. 推荐优质教程
- 视频教程或文章教程
- 教程的质量和适用人群
- 教程的优势和局限性

**推荐格式**:
```
推荐教程: [教程名称]
- 类型: [视频/文章/课程]
- 链接: [URL]
- 适用人群: [初学者/进阶者/专家]
- 优势: [教程的特点和优势]
- 预计学习时间: [X 小时/天]
```

### 3. 推荐开源项目
- 可参考的实现
- 项目的代码质量和文档质量
- 如何学习和使用这些项目？

**推荐格式**:
```
参考项目: [项目名称]
- 链接: [GitHub URL]
- 项目特点: [项目的特点和优势]
- 学习建议: [如何学习这个项目]
- 代码质量: [高/中/低]
- 文档质量: [高/中/低]
```

### 4. 推荐社区资源
- 论坛、讨论组、问答社区
- 社区的活跃度和质量
- 如何有效利用社区资源？

**推荐格式**:
```
社区资源: [社区名称]
- 链接: [URL]
- 类型: [论坛/Discord/Slack/Stack Overflow]
- 活跃度: [高/中/低]
- 适用场景: [问题求助/技术讨论/最新动态]
```

### 5. 推荐实践平台
- 在线环境、沙盒、练习平台
- 平台的特点和适用场景
- 如何使用这些平台进行学习？

**推荐格式**:
```
实践平台: [平台名称]
- 链接: [URL]
- 类型: [在线IDE/沙盒/练习平台]
- 特点: [平台的特点和优势]
- 适用场景: [快速验证/练习/项目开发]
- 费用: [免费/付费]
```

## 输出格式

```
学习资源清单:

阶段1: 基础认知
  - 官方文档: [链接 + 必读章节]
  - 推荐教程: [教程名称 + 链接]
  - 参考项目: [项目名称 + 链接]
  - 社区资源: [社区名称 + 链接]
  - 实践平台: [平台名称 + 链接]

阶段2: 进阶应用
  ...

阶段3: 项目实战
  ...
```

## 资源选择标准

### 官方文档
- ✓ 权威性高，内容准确
- ✓ 更新及时，跟随版本
- ✓ 结构清晰，易于查找
- ✗ 可能过于技术化，不适合初学者

### 优质教程
- ✓ 讲解清晰，循序渐进
- ✓ 示例丰富，易于理解
- ✓ 实践导向，注重应用
- ✗ 可能存在版本滞后问题

### 开源项目
- ✓ 真实代码，工程实践
- ✓ 可以直接运行和修改
- ✓ 社区活跃，问题可以得到解答
- ✗ 代码质量参差不齐

### 社区资源
- ✓ 问题解答快速
- ✓ 可以了解最新动态
- ✓ 可以与其他学习者交流
- ✗ 信息质量参差不齐

### 实践平台
- ✓ 无需本地环境配置
- ✓ 可以快速验证想法
- ✓ 提供练习和挑战
- ✗ 可能存在功能限制

## 设计原则

- **公开可访问**: 所有推荐资源必须是公开可访问的
- **质量优先**: 优先推荐高质量、权威的资源
- **阶段匹配**: 资源应该与学习阶段相匹配
- **多样性**: 提供多种类型的资源，满足不同学习风格
- **时效性**: 注意资源的更新时间，避免推荐过时的内容
