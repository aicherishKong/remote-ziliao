# Boundary Clarification Protocol (边界澄清协议)

## 目标
当用户的学习目标或边界不清晰时，通过结构化提问快速确定学习范围。

## 触发条件
- 用户只提供了技术名称，没有说明学习目标
- 用户的学习目标过于宽泛或模糊
- 用户的技术背景不明确
- 用户的时间预期不明确

## 标准提问清单

### 1. 核心技术确认
**问题**: "您想学习的核心技术是 [技术名称] 吗？是否包含相关的子技术或工具？"

**目的**: 确认技术边界，避免范围过大或过小

**示例**:
- "您想学习的是 Agent Skills 的使用，还是包括 MCP 协议的深度定制？"
- "您想学习的是 RAG 的基础概念，还是包括向量数据库、Embedding 模型等配套技术？"

### 2. 学习目标层级确认
**问题**: "您的学习目标是以下哪个层级？"

**选项**:
- **概念理解层**: 知道是什么、为什么需要、适用场景
- **应用实践层**: 能够使用和配置、解决实际问题
- **深度掌握层**: 能够优化和创新、设计复杂系统

**目的**: 确定学习深度，决定内容的详细程度

### 3. 技术背景确认
**问题**: "您当前的技术背景是？"

**选项**:
- **完全新手**: 没有相关技术背景
- **有基础**: 了解基本概念，但没有实践经验
- **有经验**: 有一定实践经验，想要深入学习
- **专家级**: 已经掌握核心技术，想要了解最新进展

**目的**: 确定起点，避免内容过于简单或过于复杂

### 4. 前置知识确认
**问题**: "您是否已经掌握以下前置知识？"

**示例**:
- 对于 Agent Skills: "是否了解 AI 大模型的基本概念？是否熟悉 Markdown 语法？"
- 对于 RAG: "是否了解向量数据库？是否有 Python 编程经验？"

**目的**: 确定前置知识，决定是否需要补充基础内容

### 5. 时间预期确认
**问题**: "您希望在多长时间内完成学习？"

**选项**:
- **快速入门**: 2-3 小时，了解核心概念和基本使用
- **系统学习**: 1-2 周，掌握核心技能和常见应用
- **深度掌握**: 1-2 月，能够独立设计和优化系统

**目的**: 确定学习节奏，决定内容的详细程度和实践深度

### 6. 应用场景确认
**问题**: "您学习这个技术的主要应用场景是？"

**目的**: 确定学习重点，提供针对性的案例和项目

**示例**:
- "您是想用 Agent Skills 来自动化代码审查，还是用于文档生成？"
- "您是想用 RAG 来构建企业知识库，还是用于客服机器人？"

## 澄清流程

### 步骤 1: 识别模糊点
分析用户的请求，识别哪些信息缺失或不明确。

### 步骤 2: 选择关键问题
从标准提问清单中选择 2-3 个最关键的问题。

**优先级**:
1. 核心技术确认（必问）
2. 学习目标层级确认（必问）
3. 时间预期确认（建议问）
4. 技术背景确认（根据情况）
5. 前置知识确认（根据情况）
6. 应用场景确认（根据情况）

### 步骤 3: 提出问题
以友好、简洁的方式提出问题，避免一次性问太多问题。

**示例**:
```
为了给您设计最合适的学习路径，我需要了解几个关键信息：

1. 您的学习目标是？
   - [ ] 概念理解：知道是什么、为什么需要
   - [ ] 应用实践：能够使用和配置
   - [ ] 深度掌握：能够优化和创新

2. 您希望在多长时间内完成学习？
   - [ ] 快速入门（2-3 小时）
   - [ ] 系统学习（1-2 周）
   - [ ] 深度掌握（1-2 月）

请告诉我您的选择，我会为您设计相应的学习路径。
```

### 步骤 4: 根据回答调整路径
根据用户的回答，调整学习路径的深度、广度和节奏。

## 快速决策规则

### 如果用户说"快速入门"
- 重点：基础入门视角
- 压缩：进阶案例和项目实战
- 时间：2-3 小时
- 输出：概念 + 最小示例 + 快速验证

### 如果用户说"系统掌握"
- 重点：三个视角全覆盖
- 深度：中等深度，覆盖常见场景
- 时间：1-2 周
- 输出：完整学习路径 + 案例 + 项目

### 如果用户说"深度掌握"
- 重点：三个视角全覆盖 + 扩展方向
- 深度：深入讲解，覆盖高级场景
- 时间：1-2 月
- 输出：完整学习路径 + 高级案例 + 复杂项目 + 进阶方向

## 设计原则

- **最小提问**: 只问最关键的问题，避免过度提问
- **友好表达**: 使用友好、简洁的语言，避免专业术语
- **选项明确**: 提供明确的选项，方便用户快速选择
- **灵活调整**: 根据用户的回答灵活调整后续问题
- **快速决策**: 在获得关键信息后，快速进入路径设计阶段
