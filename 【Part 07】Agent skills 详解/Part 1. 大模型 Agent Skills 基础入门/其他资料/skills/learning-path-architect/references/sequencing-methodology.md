# Sequencing Methodology (学习路径排序方法论)

## 目标
确定知识点的学习顺序，建立清晰的依赖关系。

## 执行步骤

### 1. 识别依赖关系
- **强依赖**: A 必须在 B 之前学习（例如：必须先学习 Python 基础才能学习 Flask）
- **弱依赖**: A 建议在 B 之前学习，但不是必须（例如：建议先学习 HTTP 协议再学习 RESTful API）
- **无依赖**: A 和 B 可以任意顺序学习

### 2. 识别可并行学习的知识点
- 哪些知识点可以同时学习？
- 并行学习的优势和风险
- 如何安排并行学习的时间分配？

### 3. 识别关键路径
- 必须掌握的核心链路是什么？
- 关键路径上的每个节点都不能跳过
- 关键路径决定了最短学习时间

### 4. 识别可选路径
- 根据兴趣深入的方向
- 可选路径不影响核心能力的掌握
- 但可以提供更深入的理解或更广泛的应用

### 5. 难度等级标注
使用星级标注难度（⭐~⭐⭐⭐⭐⭐）：
- ⭐: 入门级，无需前置知识
- ⭐⭐: 初级，需要基础概念
- ⭐⭐⭐: 中级，需要一定实践经验
- ⭐⭐⭐⭐: 高级，需要深入理解和综合应用
- ⭐⭐⭐⭐⭐: 专家级，需要系统掌握和创新能力

### 6. 学习时间估算
- 每个知识点的预计学习时间（小时/天/周）
- 考虑学习者的背景和学习速度
- 包含理论学习、实践练习、复习巩固的时间

## 输出格式

```
学习路径图:

阶段1: 基础认知 (预计 X 小时)
  ├─ 知识点1 [⭐] (必修, 2h)
  ├─ 知识点2 [⭐⭐] (必修, 3h)
  └─ 知识点3 [⭐] (选修, 1h)

阶段2: 进阶应用 (预计 X 天)
  ├─ 知识点4 [⭐⭐⭐] (必修, 依赖: 知识点1,2)
  ├─ 知识点5 [⭐⭐⭐] (必修, 依赖: 知识点2)
  └─ 知识点6 [⭐⭐⭐⭐] (选修, 依赖: 知识点4)

阶段3: 项目实战 (预计 X 周)
  └─ 完整项目 [⭐⭐⭐⭐⭐] (必修, 依赖: 阶段1+2)

关键路径: 知识点1 → 知识点2 → 知识点4 → 项目实战
可选扩展: 知识点3, 知识点6
```

## 设计原则

- **依赖优先**: 先学习被依赖的知识点
- **难度递进**: 从简单到复杂，符合认知规律
- **时间可控**: 每个阶段的时间预估应该合理
- **灵活性**: 提供必修和选修的区分，允许个性化学习路径
- **可验证**: 每个阶段都应该有明确的完成标准
