# Foundation Perspective (基础入门视角)

## 目标
建立正确的认知框架，理解核心概念。

## 分析维度

### 1. 概念定义
- 这个技术是什么？用一句话说清楚
- 提供清晰、准确的核心定义
- 避免使用过于技术化的术语

### 2. 问题背景
- 为什么需要这个技术？
- 它解决什么痛点？
- 在什么场景下这个技术是必要的？

### 3. 核心特性
- 这个技术的 3-5 个关键特性是什么？
- 每个特性的具体表现和价值
- 特性之间的关联关系

### 4. 对比理解
- 与类似技术的本质区别是什么？
- 什么时候选择这个技术而不是其他方案？
- 技术边界在哪里？

### 5. 基础术语
- 必须掌握的 10 个核心术语
- 每个术语的准确定义
- 术语之间的关系和层次

### 6. 最小可用示例
- 一个 5 分钟能跑通的 Hello World
- 示例应该简单但完整
- 包含运行验证步骤

## 输出格式

```
基础入门路径:
1. 概念建立: [核心定义 + 类比理解]
2. 痛点理解: [真实场景 + 对比方案]
3. 核心特性: [特性1, 特性2, 特性3]
4. 术语体系: [术语表 + 释义]
5. 快速上手: [最小示例 + 运行验证]
```

## 设计原则

- **由浅入深**: 从最基础的概念开始，逐步深入
- **类比优先**: 使用学习者熟悉的概念进行类比
- **实践验证**: 每个概念都应该有可验证的示例
- **避免跳跃**: 不要假设学习者已经掌握相关背景知识
